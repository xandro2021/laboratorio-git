IF-4101: Lenguajes para Aplicaciones Comerciales

Laboratorio 2: Tutorial de Git y GitHub

UCR - Sede del Caribe, Recinto de Siquirres

Guía técnica para la implementación de control de versiones y flujos de trabajo profesionales en entornos académicos.

Contenido del Curso

1. Configuración 2. Estructura de Proyecto 3. Inicialización 4. Primer Commit 5. Gestión de Ramas 6. Modificaciones 7. Fusión (Merge) 8. Comparación (Diff) 9. Reversión (Reset) 10. Recuperación 11. Exclusiones (.gitignore) 12. Integración Remota 13. Feature Branching 14. Pull Requests

01. Configuración de Identidad

Requisito

Establece tu identidad oficial para asegurar la trazabilidad de los cambios en el repositorio del curso.

git config --global user.name "Tu Nombre Completo"
git config --global user.email "tu.nombre@ucr.ac.cr"

02. Estructura del Proyecto

Crea un espacio de trabajo organizado para el Laboratorio 2.

mkdir ucr-laboratorio-2
cd ucr-laboratorio-2

03. Inicialización del Repositorio

Inicia el rastreo de versiones creando la estructura interna de Git.

git init

04. Primer Commit de Control

Registra el estado inicial de tu documentación del curso.

touch README.md
git add .
git commit -m "UCR Lab 2: Registro inicial de documentación"

05. Ramas de Desarrollo

Separa las nuevas funcionalidades de la rama principal para mantener la estabilidad del proyecto IF-4101.

git checkout -b desarrollo
school
menu_book

Estándar Académico: Commits Descriptivos

En el curso IF-4101 se valora la claridad semántica. Cada commit debe iniciar con un verbo en infinitivo o imperativo que resuma el cambio técnico realizado. Evita mensajes genéricos.

06. Modificación y Registro

Actualiza el contenido técnico y regístralo en la rama de desarrollo.

echo "Laboratorio 2 - UCR Siquirres" >> index.html
git add index.html
git commit -m "Add header to index.html with campus reference"

07. Fusión de Cambios (Merge)

Incorpora el progreso validado a la rama maestra del curso.

git checkout main
git merge desarrollo

08. Análisis de Diferencias (Diff)

Verifica las discrepancias entre estados antes de consolidar cambios.

git diff main desarrollo
Módulo 09

Reversión Local

Retorna al estado anterior del proyecto. Utiliza con precaución en entornos colaborativos de la UCR.

git reset --hard HEAD~1

10. Recuperación de Archivos

Restaura recursos académicos eliminados accidentalmente.

git checkout -- nombre-archivo.txt

11. Exclusión de Archivos (.gitignore)

Evita subir dependencias pesadas o credenciales al repositorio de la Universidad.

echo "node_modules/" >> .gitignore
echo ".env" >> .gitignore

Integración con GitHub

12. Sincronización Remota

Vincula tu repositorio local con el alojamiento en la nube.

git remote add origin https://github.com/tu-usuario/ucr-lab-2.git
git push -u origin main

13. Rama de Funcionalidad Específica

Demostración de flujo de trabajo colaborativo para el curso.

git checkout -b feature/login-module
git push origin feature/login-module

14. Pull Request y Validación

Proceso formal de revisión de código para entregas académicas.

  1. Acceda a su repositorio en GitHub → Pull Requests → New Pull Request.
  2. Compare la rama base main con la rama feature/login-module.
  3. Documente sus cambios y proceda al Merge tras la validación.
